# Журнал изменений (Changelog)

Все заметные изменения в этом проекте будут документироваться в этом файле.

## [3.4.8] - 2026-06-13

### Added
- **Взаимная приостановка записи и TTS**:
  - При старте воспроизведения TTS (в `voice-core.js`) теперь отправляется событие `tts-playing-started`, по которому Капсула автоматически приостанавливает диктофонную запись, если она запущена.
  - При старте или продолжении записи в Капсуле отправляется событие `recording-started`, по которому плеер TTS автоматически встает на паузу.

### Fixed
- **Сохранение и восстановление буфера обмена с изображениями**:
  - Расширен механизм бэкапа буфера обмена: теперь помимо текста сохраняются и восстанавливаются стандартизированные форматы картинок Windows — `CF_DIB` (8) и `CF_DIBV5` (17).
  - Добавлен цикл ожидания (`retry-loop` из 15 попыток с задержкой 15 мс) для борьбы с кратковременной блокировкой системного буфера ОС Windows.
  - Добавлено логирование операций в `debug_clipboard.log` и защита от затирания буфера в ноль при пустом бэкапе.

### Changed
- **Оптимизация префетча Edge TTS**:
  - Внедрен единый метод `getPrefetchDelay(index)` (принцип DRY), устранивший рассинхронизацию задержек между планированием и циклом ожидания `while`.
  - Увеличено окно мгновенной предзагрузки: теперь для текущего и ближайших 3 предложений (`distance <= 3`) задержка составляет `150 мс`. Это гарантирует защиту от сбоев Windows Schannel (TLS error -2146893008) при частых WebSocket-запросах.
  - Оптимизированы фоновые задержки для остальных предложений (от `400` до `1400 мс` с учетом проигрывания/паузы) для плавного опережения чтения без риска банов.
  - Убрана избыточная проверка `audioElement.paused` при автозапуске воспроизведения для устранения зависания на стыках предложений.
  - Внедрено подробное логирование жизненного цикла TTS (`this.log`) в файл `tts_errors.log` для посекундного мониторинга работы буферизации.

## [3.4.7] - 2026-06-12

### Added
- **Среда ИИ-обработки («Новый запрос»)**:
  - Добавлено стильное интерактивное модальное окно `#new-request-modal` для выполнения ИИ-запросов на базе выбранных источников и пресетов.
  - Реализован выбор источников: `Первоисточник (Аудиозапись)`, `Транскрипция (Исходный текст)` или любой из ранее созданных результатов `ИИ-обработки` (для реализации цепочек промптов).
  - Поддерживаются пресеты: `Суть`, `Задачи`, `Улучшенный текст`, `Письмо` и `Свой собственный промпт` (динамически отображающий текстовое поле ввода).
  - Автоматическая генерация названия результата по маске `[Пресет] ([Источник])` с возможностью ручной перезаписи.
  - Асинхронная Tauri-команда `process_ai_request` на бэкенде (Rust), поддерживающая передачу как аудиозаписей в Base64 (inlineData), так и текстовых фрагментов для оптимизации трафика Gemini API, а также отказоустойчивые цепочки фоллбэк-моделей.
  - Автоматическое переключение вида дашборда на вновь созданный результат сразу после генерации.

## [3.4.6] - 2026-06-10

### Added
- **3-позиционная схема управления**: Внедрены слайдеры выбора режима работы («Выкл» / «Только хоткеи» / «Все включено») для Капсулы, Виджета озвучки и OCR-переводчика.
- **Временное появление по хоткеям**: В режиме «Только хоткеи» (1) окна временно отображаются на экране во время выполнения операции и плавно скрываются по окончании.
- **Сохранение и восстановление геометрии окон**: Все окна (`main`, `capsule`, `widget`, `prompt`, `result`) запоминают свои координаты (и размеры, где применимо) в `localStorage`.
- **Проверка смены мониторов**: На старте геометрия окон сверяется со списком активных экранов через `availableMonitors()`. Если окно было на отключенном экране, оно сбрасывается в штатные координаты основного монитора.
- **Автозапуск с Windows**: В раздел системных настроек дашборда добавлен тумблер автозапуска, регистрирующий/удаляющий запись автозагрузки в реестре Windows (`HKCU\Software\Microsoft\Windows\CurrentVersion\Run`) с флагом `--minimized`.
- **Умное поведение автозапуска**: При старте ОС с флагом `--minimized` дашборд остается скрытым в трее, если был закрыт пользователем, но все активные виджеты (в режиме «Все включено») запускаются и отображаются на своих местах на экране.

### Changed
- Окна `main`, `capsule` и `widget` переведены в режим невидимых по умолчанию (`"visible": false` в `tauri.conf.json`) для исключения мигания интерфейса при старте системы.

### Fixed
- **Санитаризация Markdown**: Исправлена очистка служебных символов разметки перед отправкой текста в Edge TTS.
- Полностью исправлен и запущен E2E-тест Playwright (`tests/sinc-pro.spec.js`), все 15 тестов проходят успешно.
