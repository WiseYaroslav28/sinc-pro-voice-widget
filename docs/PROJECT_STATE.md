# Реестр Состояний SINC PRO

Этот документ описывает актуальное состояние проекта, принятые технические решения и архитектуру. **ИИ-Агенты обязаны сверяться с этим файлом перед любыми изменениями.**

## 🏗 Архитектура

Проект построен на связке:
* **Бэкенд (Rust / Tauri)**: Отвечает за системные хуки клавиатуры (WinAPI), работу с буфером обмена, управление DPI мониторов, хеширование токенов (Edge TTS) и сетевые запросы к ИИ-моделям (Gemini) в обход CORS.
* **Фронтенд (HTML/CSS/JS)**: Vanilla JS. Вся стилизация построена на кастомной дизайн-системе (Stitch) без использования тяжелых фреймворков.

## 🛠 Принятые Технические Решения (Golden Rules)
1. **Zero Trust Policy**: Не доверять сообщениям в старых коммитах, пока код не проверен вручную.
2. **WinAPI vs Web APIs**: Из-за ограничений WebView (напр. доступ к буферу обмена без фокуса окна), критически важные системные взаимодействия (чтение буфера, позиции курсора) переносятся на Rust-бэкенд с использованием `windows-sys` или `winapi`.
3. **Логирование**: Для отладки Edge TTS во фронтенде временно используем `console.error` вместо `console.warn`.
4. **Свободные координаты капсулы**: В `resizeToState` убраны ограничения `Math.max(0, ...)`, чтобы капсула могла уходить в отрицательные координаты на мультимониторных системах.
